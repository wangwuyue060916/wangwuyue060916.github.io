钢铁雄心 4 决议组制作教程

一、决议组基础概念

（一）核心组成

决议组由两部分构成：

  1. 决议组（Category）：用于分类管理具体决议，类似 “百宝袋”（如 “经济政策”“军事行动”）。
  2. 具体决议（Decision）：包含实际效果的执行代码（如 “修建铁路”“宣战”），隶属于某个决议组。

（二）文件路径

  • 决议组注册文件：common/decisions/categories/[文件名].txt
  • 具体决议文件：common/decisions/[文件名].txt

（二）关键参数说明

参数名称 作用 示例值
category 归属的决议组 ID category = my_mod_economic_decisions
available 决议可点条件（支持逻辑运算符AND/OR/NOT） available = { country_size > 5 AND has_idea = "central_planning" }
effect 决议执行效果（核心部分，支持国策、事件、资源修改等） effect = { add_equipment_to_stockpile = { type = "infantry_equipment" amount = 2000 } }
cost 执行决议消耗的资源（政治点、人力等） cost = { political_power = 30 }
icon 决议图标（需在.gfx文件中注册路径） icon = "GFX_decision_railway"

五、常见问题与交流

（一）报错排查

  1. 决议组未显示：
    • 检查决议组 ID 是否拼写错误（区分大小写）。
    • 确认本地化文本键名与代码中的 ID 一致。
  2. 效果未触发：
    • 使用控制台命令 debug_decisions 强制显示所有决议，排查 available 条件是否满足。
    • 通过 event debug 1 输出决议执行日志。
